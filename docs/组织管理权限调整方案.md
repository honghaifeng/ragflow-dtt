# 组织管理权限调整方案

## Context

当前状态：Organization、OrgMember、KbPermission 三张表已创建，后端 API（org_app.py、kb_permission_app.py）已开发，用户端组织管理页面（`/organization`）已开发，管理后台团队管理页面已对接 Organization 表。

**问题**：当前用户端导航栏对所有用户无条件显示"组织管理"入口，但实际上：
- 组织的创建、成员管理等操作应该由**管理后台**（系统管理员）统一管控
- 用户端只有被赋予 owner/admin 角色的用户才应看到"组织管理"
- editor/viewer 角色的用户不需要组织管理入口

**目标**：调整权限分工——管理后台做全局组织管理，用户端按角色动态显示。

---

## 一、权限分工

| 操作 | 管理后台（系统管理员） | 用户端（owner） | 用户端（admin） | 用户端（editor/viewer） |
|------|:---:|:---:|:---:|:---:|
| 创建组织 | ✅ | ❌ | ❌ | ❌ |
| 删除组织 | ✅ | ✅（自己的） | ❌ | ❌ |
| 查看所有组织 | ✅ | ❌ | ❌ | ❌ |
| 管理成员（添加/移除） | ✅ | ✅ | ✅ | ❌ |
| 修改成员角色 | ✅ | ✅ | ❌ | ❌ |
| 生成邀请链接 | ✅ | ✅ | ✅ | ❌ |
| 管理知识库权限 | ✅ | ✅ | ✅ | ❌ |
| 查看组织信息 | ✅ | ✅ | ✅ | ❌（不显示入口） |
| 使用知识库 | - | ✅ | ✅ | ✅ |

## 二、后端 API 调整

### 2.1 新增角色查询接口 `org_app.py`

| 接口 | 方法 | 说明 |
|------|------|------|
| `/v1/org/my_role` | GET | 返回当前用户在所有组织中的最高角色（用于前端判断是否显示导航入口） |

```
响应示例:
{
  "data": {
    "has_org_admin": true,   // 是否在任意组织中为 owner 或 admin
    "roles": [
      {"org_id": "xxx", "org_name": "电力运维部", "role": "owner"},
      {"org_id": "yyy", "org_name": "信息中心", "role": "admin"}
    ]
  }
}
```

### 2.2 修改创建组织权限 `org_app.py`

现状：`/v1/org/create` 任何登录用户都能创建组织。
调整：只允许管理后台调用创建组织（通过管理后台专用 API key 或角色校验）。

**方案**：在 `create_org()` 中增加 `is_superuser` 检查。管理后台通过其专用 JWT 或请求头标识来调用。普通用户端调用返回 403。

### 2.3 管理后台新增管理员 API

在 `management/server/routes/teams/routes.py` 中确保以下功能完整：
- 创建组织
- 删除组织
- 添加/移除成员
- 修改成员角色
- 查看所有组织（不限于自己创建的）

## 三、用户端（React）调整

### 3.1 导航栏动态显示

**文件**: `web/src/layouts/components/header/index.tsx`

现状：`/organization` 无条件出现在 `tagsData` 数组中。
调整：
1. 页面加载时调用 `/v1/org/my_role` 接口
2. 仅当 `has_org_admin === true`（用户在某个组织中是 owner 或 admin）时，才在 `tagsData` 中包含 `{ path: '/organization', name: t('organization'), icon: TeamOutlined }`
3. editor/viewer 用户看不到这个入口

### 3.2 路由守卫

**文件**: `web/src/routes.ts`

给 `/organization` 路由增加权限校验：
- 未登录 → 跳转登录
- 已登录但不是任何组织的 owner/admin → 跳转首页或显示无权限提示

### 3.3 组织管理页面内容调整

**文件**: `web/src/pages/organization/index.tsx`

- 移除"创建组织"按钮（创建组织只在管理后台）
- 保留：成员管理、邀请链接、知识库权限设置（仅 owner/admin 可见的组织）
- 页面标题改为"我的组织"，展示用户作为 owner/admin 的组织列表

## 六、实施顺序

1. **后端** — org_app.py 增加 `/my_role` 接口 + `/create` 权限校验
2. **用户端** — header 动态显示 + 路由守卫 + 页面调整
3. **管理后台** — 改名 + 功能验证
4. **部署** — 同步后端代码到容器 + 重新 build 前端 + 重启

## 七、验证方式

1. 管理后台创建组织"电力运维部"，添加用户 A（admin）和用户 B（viewer）
2. 用户 A 登录用户端 → 导航栏显示"组织管理"→ 可管理成员和知识库
3. 用户 B 登录用户端 → 导航栏**不显示**"组织管理"→ 但能正常使用被授权的知识库
4. 普通新注册用户 → 导航栏不显示"组织管理"
5. 管理后台可看到所有组织、创建/删除组织、管理任意组织成员
