# 电力行业知识图谱 — RAGFlow GraphRAG 能力分析

> 分析日期: 2026-04-14
> RAGFlow版本: v0.17.2

---

## 一、现有 GraphRAG 能力概述

RAGFlow 的 GraphRAG 模块（位于 `graphrag/`）是一个 **基于 LLM 提示词驱动的通用知识图谱抽取引擎**，核心流程如下：

1. **文档分块** — 将文档按配置切分为若干 chunk
2. **实体/关系抽取** — 每个 chunk 通过 LLM（如 DeepSeek、Qwen等）进行实体识别和关系抽取
3. **实体消歧** — 对重复/相似实体进行合并（Entity Resolution）
4. **社区发现** — 使用 Leiden 算法对图进行社区检测
5. **图谱存储** — 存入 knowledge_graph 集合，支持图谱检索

### 关键特性

| 特性 | 说明 |
|------|------|
| 抽取方式 | LLM 提示词驱动，非传统 NER 模型 |
| 实体类型 | 可自定义，默认: organization, person, geo, event, category |
| 抽取方法 | Light（轻量）/ General（通用，含社区发现） |
| 实体消歧 | 支持，基于 LLM 判断相似实体是否为同一实体 |
| 社区检测 | Leiden 算法，自动发现关联紧密的实体群组 |
| 并行处理 | 支持多 chunk 并行调用 LLM |

## 二、电力行业适用性分析

### 2.1 可以做到的（开箱即用）

1. **自定义实体类型**：GraphRAG 的 `entity_types` 参数完全可配置，可以设置为电力行业专属类型：
   - 设备、故障类型、保护装置、安全措施、操作规程
   - 电压等级、技术参数、组织机构、法规标准

2. **通用关系抽取**：LLM 能从文本中识别实体间的关系描述（如"属于"、"保护"、"导致"等），并以三元组形式存储

3. **实体消歧**：对于同一设备的不同称谓（如"110kV断路器"与"高压断路器"），消歧模块可尝试合并

4. **社区发现**：自动发现关联紧密的实体群组（如某一类型故障关联的所有保护装置）

### 2.2 存在的局限

1. **抽取质量依赖 LLM**：
   - 通用 LLM 对电力行业术语理解深度有限
   - 可能将"接地故障"误识别为地理位置相关实体
   - 专业缩写（如 CT、PT、SF6）可能无法正确识别

2. **Prompt 模板为通用设计**：
   - `GRAPH_EXTRACTION_PROMPT` 中的示例是通用领域（人物、科技、任务等）
   - 缺少电力行业的 few-shot 示例
   - 关系类型没有电力专业约束

3. **无领域本体约束**：
   - 没有预定义的电力设备分类体系
   - 没有标准的故障-原因-措施关联模型
   - 无法验证抽取结果是否符合电力行业知识体系

4. **缺少结构化信息抽取**：
   - 无法从表格中抽取设备参数（额定电压、额定电流等）
   - 无法识别文档中的编号体系（如条款编号 4.2.3.1）
   - 对规程的层级结构理解有限

## 三、测试方案

### 3.1 测试文档

| 文档 | 类型 | 预期抽取内容 |
|------|------|-------------|
| 二十五项反措 | 安全规程 | 设备类型、事故类型、防范措施、技术要求 |
| 电网运行规则 | 运行规程 | 调度术语、操作流程、电压等级、设备操作 |
| 供电营业规则 | 营业规程 | 业务流程、收费标准、用户分类、法规引用 |

### 3.2 配置的实体类型

```
设备, 故障类型, 保护装置, 安全措施, 操作规程, 电压等级, 技术参数, 组织机构, 法规标准
```

### 3.3 评估维度

1. **准确率**：抽取的实体是否确实是该类型
2. **召回率**：文档中的关键实体是否被覆盖
3. **关系质量**：抽取的关系是否有实际意义
4. **消歧效果**：同一实体的不同表述是否被正确合并

## 四、增强建议

### 4.1 短期优化（不改代码）

1. **优化实体类型配置**：根据测试结果调整实体类型列表，去掉效果差的类型
2. **选择更强的LLM**：使用电力行业理解较好的模型（如经过行业数据微调的模型）
3. **分批处理**：不同类型的文档使用不同的实体类型配置

### 4.2 中期优化（改Prompt）

1. **定制 GRAPH_EXTRACTION_PROMPT**：
   - 添加电力行业 few-shot 示例
   - 定义标准关系类型（如"保护→设备"、"措施→故障"等）
   - 增加电力术语缩写对照表

2. **定制消歧 Prompt**：
   - 添加电力设备同义词规则
   - 设备型号的标准化规则

### 4.3 长期优化（自研增强）

1. **领域 NER 模型**：训练电力行业专用的命名实体识别模型，替代或辅助 LLM 抽取
2. **本体知识库**：构建电力行业本体（设备分类、故障树、保护配置体系），用于约束和验证抽取结果
3. **规则引擎**：针对特定文档格式（如反措条款结构），开发规则化抽取逻辑
4. **表格解析增强**：专门处理设备参数表、技术指标表等结构化数据

---

## 五、结论

RAGFlow 的 GraphRAG **具备基础的电力行业知识图谱抽取能力**，通过自定义实体类型可以初步实现从电力专业文档中抽取实体和关系。但抽取质量主要取决于底层 LLM 对电力行业术语的理解程度，且缺少领域本体约束。

**建议路径**：先用现有能力跑测试 → 评估抽取质量 → 根据结果决定优化 Prompt 还是需要自研 NER 模型。
